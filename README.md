# Tmunot (תמונות)

A self-contained, high-performance, multi-tenant photo and video gallery application written in Zig. 

The frontend assets (HTML, minified CSS/JS, and fonts) are compiled directly into a single lightweight binary, yielding zero-dependency static page delivery. Media compression and extraction pipelines are executed asynchronously in the background.

---

## Features

- ⚡ **High-Performance Image Engine**: Pipelined image decoding, scaling, and thumbnail generation using native `libvips` bindings.
- 🎬 **Video Hosting & Previews**: Probes and processes uploaded video clips. Generates a 3-second silent AV1-encoded looping preview (transcoded with `ffmpeg`) that plays on hover, plus a static poster thumbnail.
- 📁 **Chronological Album Management**:
  - Create, view, and manage albums.
  - Add or delete photos from albums individually or in bulk.
  - Displays media in a justified grid sorted chronologically (oldest first).
- 👥 **Multi-Tenant Architecture**: Stores records in independent SQLite databases per user (`databases/{username}.db`) utilizing Write-Ahead Logging (WAL) mode for maximum write concurrency.
- ⚙️ **Admin Dashboard**: Web interface for managing user registrations and dynamically tuning directory configurations at runtime.
- 🔒 **Security**:
  - User passwords hashed using memory-hard Argon2id.
  - Stateful JWT-based sessions with dual-tier signing key retrieval (persisted to a local secure binary file `jwt_secret.bin` or loaded from environment variables).
  - Fine-grained per-request arena allocators to guarantee thread-safe operation and prevent memory leaks.

## System Requirements

### Build-Time Dependencies
To build the project, the system must have:
* **Zig Compiler**: Standard Zig compiler toolchain.
* **Python 3**: For build-time stylesheet and JavaScript minification and HTML inlining.
* **Development Packages** (Debian/Ubuntu):
  * `libvips-dev`
  * `libexif-dev`
  * `libsqlite3-dev`

### Runtime Dependencies
At runtime, the binary requires the following system dependencies:
* **Libraries**: `libvips`, `libexif`, `libsqlite3`.
* **CLI Tools**: `ffmpeg` and `ffprobe` (must be available in your system `$PATH`).

## Getting Started

### 1. Build from Source
Compile the executable:
```bash
zig build -Doptimize=ReleaseFast
```
The compiled binary will be placed at `./zig-out/bin/tmunot`.

### 2. Configure the Server
Create a `config.json` configuration file in the working directory of the application:
```json
{
  "backend": "libvips",
  "quality": 90,
  "gallery_thumbnail_height": 165,
  "input_directory": "./input",
  "db_dir": "./databases",
  "originals_dir": "./photos/originals",
  "previews_dir": "./photos/previews",
  "thumbnails_dir": "./photos/thumbnails",
  "hover_previews_dir": "./photos/hover_previews",
  "outputs": [
    {
      "name": "previews",
      "target_width": 1200,
      "target_height": 1200
    },
    {
      "name": "thumbnails",
      "target_width": 600,
      "target_height": 600
    }
  ]
}
```

### 3. Run the Server
Launch the server from the directory containing `config.json`:
```bash
./zig-out/bin/tmunot
```
By default, the server binds to port `3001` on all interfaces (`0.0.0.0:3001`).

* **First Startup**: If `users.json` is missing or empty, the application will automatically bootstrap a default administrator account with username `admin` and password `admin`. Change these credentials immediately via the Admin panel.

## Directory Layout & State Configuration

Ensure the following directories and files are persisted (for example, in Docker mounts or VM volumes) to prevent data loss on container rebuilds:

```text
├── tmunot                  # Compiled binary (Stateless)
├── config.json             # Configuration settings (Read-only)
├── users.json              # Registered users & Argon2id hashes (Stateful)
├── jwt_secret.bin          # Autogenerated JWT signing key (Stateful)
├── databases/              # Per-user SQLite databases (Stateful)
└── photos/                 # Uploaded originals & generated thumbnails (Stateful)
```
